# 設計ナビゲーター

設計の全体像を把握し、次に何をすべきかをガイドします。

## 作業プロセス

### Step 1: 全設計書の状態を確認
1. `設計/01_要件定義.md` を読み込み、完成度を評価
2. `設計/02_基本設計.md` を読み込み、完成度を評価
3. `設計/03_詳細設計.md` を読み込み、完成度を評価

### Step 2: 進捗レポート
以下の形式で報告:

```
## 設計進捗レポート

| フェーズ | 状態 | 完成度 | 次のアクション |
|---------|------|--------|--------------|
| 要件定義 | ✅完了 / 🔧作業中 / ⬜未着手 | XX% | ... |
| 基本設計 | ✅完了 / 🔧作業中 / ⬜未着手 | XX% | ... |
| 詳細設計 | ✅完了 / 🔧作業中 / ⬜未着手 | XX% | ... |
```

### Step 3: 推奨アクション
- 次に実行すべきスキルコマンドを具体的に提案:
  - `/requirements 作成` — 要件定義を作成
  - `/requirements レビュー` — 要件定義をレビュー
  - `/basic-design 作成` — 基本設計を作成
  - `/basic-design 作成 DB設計` — DB設計に絞って作成
  - `/detailed-design 作成` — 詳細設計を作成
  - `/detailed-design 作成 患者入力画面` — 特定画面の詳細設計

### Step 4: 整合性チェック
- 設計書間の矛盾がないか簡易チェック
- 問題があれば `/fact-check 設計書` の実行を推奨
