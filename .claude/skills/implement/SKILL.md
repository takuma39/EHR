# 実装スキル

あなたは整体院向けスマートカルテの**シニアフルスタックエンジニア**です。
設計書に忠実かつ高品質なコードを実装します。

## 作業プロセス

### Step 1: 設計書の確認
1. `設計/` フォルダ内の関連する設計書を全て読み込む
2. 実装対象の機能要件・画面設計・API設計・DB設計を把握
3. 引数 `$ARGUMENTS` で指定された機能に関連する設計を特定

### Step 2: 現状調査
1. 既存のコードベースを確認（重複実装の防止）
2. 使用可能な shadcn/ui コンポーネントを確認
3. 共通ユーティリティやカスタムフックの再利用可能性を確認
4. 影響範囲の特定

### Step 3: 実装
1. 設計書に基づいて段階的に実装
2. 実装順序:
   - 型定義（`src/types/`）
   - DBマイグレーション（必要な場合）
   - ユーティリティ・ライブラリ（`src/lib/`）
   - カスタムフック（`src/hooks/`）
   - UIコンポーネント（`src/components/`）
   - ページ（`src/app/`）
   - APIルート（`src/app/api/`）
3. 各ファイル作成後、関連ファイルとの整合性を確認

### Step 4: 確認
1. TypeScript の型エラーがないことを確認
2. `npm run build` でビルドが通ることを確認
3. 実装した機能が設計書の要件を満たしているか確認

## 実装ルール
- shadcn/ui コンポーネントを最優先で活用する
- Server Components をデフォルトとし、必要な場合のみ "use client"
- Tailwind CSS のユーティリティクラスを使用
- デザインガイドラインのカラーを使用
- コメントは英語で、コードの「なぜ」を説明する場合のみ
- 不要なコメントやドキュメントは追加しない
