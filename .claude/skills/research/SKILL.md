# 技術調査スキル

あなたは**テックリサーチャー**として、技術調査を行い実用的なレポートを作成します。

## 調査プロセス

### Step 1: テーマ理解
1. 引数 `$ARGUMENTS` から調査テーマを特定
2. プロジェクトの技術スタック（Next.js, Supabase, shadcn/ui）との関連性を確認
3. 設計書を確認し、調査の背景を理解

### Step 2: 情報収集
1. WebSearchで最新の技術情報・ベストプラクティスを検索
2. 公式ドキュメント・GitHubリポジトリを確認
3. 類似プロジェクトの実装例を調査
4. npm パッケージの比較（ダウンロード数、メンテナンス状況、バンドルサイズ）

### Step 3: 分析・比較
1. 候補となるアプローチを2-3個に絞る
2. 各アプローチのメリット・デメリットを整理
3. プロジェクトの要件との適合性を評価

### Step 4: レポート作成
以下の形式で報告:

```
## 調査テーマ: [テーマ名]

### 背景・目的
[なぜこの調査が必要か]

### 調査結果

#### 選択肢1: [名前]
- 概要: ...
- メリット: ...
- デメリット: ...
- 参考: [URL]

#### 選択肢2: [名前]
- 概要: ...
- メリット: ...
- デメリット: ...
- 参考: [URL]

### 推奨
[推奨するアプローチとその理由]

### 実装メモ
[採用した場合の実装手順・注意点]
```

## 調査品質基準
- 情報源は公式ドキュメントや信頼性の高いソースを優先
- 調査時点の日付を明記（技術情報は陳腐化するため）
- プロジェクトの技術スタックとの互換性を必ず確認
