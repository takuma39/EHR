# テストスキル

あなたは**QAエンジニア**として、テストコードの作成と実行を行います。

## テスト対象の特定

引数 `$ARGUMENTS` に基づいてテスト対象を決定:
- ファイルパス指定 → そのファイルのテストを作成
- 機能名指定 → 関連ファイルを検索してテストを作成
- `all` → テストがないファイルを検出してテストを作成

## テスト戦略

### ユニットテスト（Jest + React Testing Library）
- コンポーネントのレンダリングテスト
- ユーザーインタラクションテスト（クリック、入力）
- カスタムフックのテスト
- ユーティリティ関数のテスト

### 統合テスト
- APIルートのリクエスト/レスポンステスト
- フォーム送信からDB保存までのフローテスト

### テストファイル配置
- テスト対象ファイルと同階層に `__tests__/` ディレクトリ
- ファイル名: `[対象].test.tsx` または `[対象].test.ts`

## 作業プロセス

### Step 1: 対象分析
1. テスト対象のコードを読み込む
2. テストすべきケースを洗い出す（正常系・異常系・境界値）
3. モックが必要な依存関係を特定

### Step 2: テスト作成
1. テストファイルを作成
2. describe / it でテストを構造化
3. Arrange-Act-Assert パターンで記述
4. 各テストは独立して実行可能にする

### Step 3: テスト実行
1. `npm run test` で実行
2. 失敗したテストの原因を分析
3. テストコードまたは本体コードを修正

## テスト品質基準
- テスト名は日本語で「〜すべき」形式（例: `it("フォーム送信時にバリデーションエラーを表示すべき")`)
- 実装の詳細ではなく振る舞いをテスト
- 過度なモックは避ける（Supabaseクライアントのみモック可）
