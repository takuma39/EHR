# フロントエンド規約

## コンポーネント設計
- shadcn/ui コンポーネントを最優先で活用する（自作する前に既存を確認）
- Server Components をデフォルトとし、インタラクションが必要な場合のみ `"use client"` を付与
- 1ファイル1コンポーネントを原則とする

## スタイリング
- Tailwind CSS のユーティリティクラスを使用（インラインstyle禁止）
- デザインガイドラインのカラーを `tailwind.config.ts` で定義して使用
- レスポンシブ対応: モバイルファーストで記述（`sm:`, `md:`, `lg:`）

## アクセシビリティ
- インタラクティブ要素には `aria-label` を付与
- フォーム要素には `<label>` を紐付ける
- タブレット操作を考慮し、タッチターゲットは最低 44x44px

## 画面遷移
- Framer Motion で画面遷移アニメーションを実装
- `AnimatePresence` でページ切り替え時のフェードを統一

## 患者モード固有
- 大きめのフォントサイズ（最低16px）
- タッチ操作に最適化した大きなボタン
- ステップ形式のウィザードUI
